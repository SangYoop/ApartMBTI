# 아파트 VS 투표 (커뮤니티) 기능 구현 계획

사용자가 입력한 검색 조건을 기반으로 아파트 정보를 호출, 다중 선택(최대 4개)하여 투표 및 댓글 토론 등을 지원하는 커뮤니티 기능 구현 계획입니다.

## 1. Supabase DB 설계 (추가/수정 필요 테이블)

기존에 존재하는 아파트 기본 정보(Supabase 데이터)와 연결되는 투표(Poll) 및 커뮤니티용 테이블 구조를 아래와 같이 신설합니다.

### A. 투표 본체 테이블 (`polls`)
투표 주제와 작성 정보를 관리합니다.
* `id` (uuid) : 투표 고유 ID (Primary Key)
* `title` (text) : 투표 제목 (예: "신혼부부 실거주 고민중입니다, 어디가 나을까요?")
* `created_at` (timestamp) : 생성일시
* `created_by` (uuid) : 작성자 ID (향후 회원가입 연동 고려, 우선은 무명/익명 처리 가능)

### B. 투표 선택지(아파트) 테이블 (`poll_options`)
각 투표 항목(최대 4개)과 그와 연결된 아파트 데이터를 관리합니다.
* `id` (uuid) : 선택지 고유 ID (PK)
* `poll_id` (uuid) : 어떤 투표(`polls.id`)에 속해 있는지 (외래키, cascade)
* `apartment_id` (uuid/text) : 투표 항목으로 선택된 아파트 고유 ID (기존 아파트 테이블의 외래키)
* `vote_count` (int) : 현재 득표 수 (기본값: 0)

### C. 댓글 테이블 (`poll_comments`)
특정 투표글에 달린 댓글과 토론 내용을 저장합니다.
* `id` (uuid) : 댓글 고유 ID (PK)
* `poll_id` (uuid) : 어떤 투표(`polls.id`)에 소속된 댓글인지 (외래키, cascade)
* `content` (text) : 댓글 내용
* `created_at` (timestamp) : 생성일시
* `author_name` (text) : 작성자 닉네임 (익명 기반)

## 2. 프론트엔드 구조 (Next.js App Router 기준)

단순한 줄글보다는 인스타그램이나 블라인드와 같이 시각적인 '피드(Feed) 형태'로 구성하여 사용자의 참여율과 가독성을 높입니다.

### A. 주요 라우팅 (페이지) 구조
* **`/polls` (투표 피드 메인)**: 
  * 전체 투표를 리스트 업하는 메인 게시판.
  * 각 활성 투표는 큰 피드 카드 형태로 노출되어, 피드에서 바로 썸네일(아파트 요약)과 함께 어떤 투표가 진행중인지 확인 가능.
* **`/polls/create` (투표 생성 페이지)**: 
  * 아파트를 검색하고 투표를 개설하는 화면.
  * 단계별 UX: `1단계: 제목 작성` ➔ `2단계: 아파트 검색 및 선택 (최대 4개)` ➔ `3단계: 등록`
* **`/polls/[id]` (투표 상세 및 토론 페이지)**: 
  * 개별 투표 페이지로, 클릭 시 즉각적으로 % 결과가 반영되는 애니메이션 뷰.
  * 하단에 댓글(`poll_comments`) 컴포넌트를 붙여 아파트별 세부 의견 토론장 제공.

### B. 핵심 React / UI 컴포넌트 목록
* **`PollFeedCard`**: `/polls`에서 사용하는 UI. 제목, 참가자수, 그리고 투표로 올라온 아파트들(카드형태)의 썸네일/명칭을 리스트 형태로 출력합니다.
* **`ApartmentSearch`**: `/polls/create`에서 사용. 
  * 시도/시군구/동리 + 단지명 검색 지원 (입력시 Supabase 실시간 쿼리로 자동완성)
* **`ApartmentSummaryCard`**: 기존 작성해두신 '기본정보가 요약되는 카드'. 선택된 아파트의 핵심 스펙(준공일, 세대수, 주차대수 등)만 쏙 뽑아 보여주는 예쁜 컴포넌트입니다.
* **`PollVotingBoard`**: `/polls/[id]` 상세 영역에 들어갈 실질적 투표 UI. Framer Motion 효과를 줘서 클릭 시 V막대그래프가 확 채워지는 동적인 느낌을 줍니다.

## 3. 사용자 경험 (UX) 플로우 정리

1. **투표 생성 (작성자)** 
   + 버튼 클릭 ➔ "어느 아파트를 고민중이신가요?" ➔ 검색 입력 ➔ `ApartmentSummaryCard` 노출 ➔ [+ 추가] 버튼으로 또 다른 아파트 검색 후 담기 (Max 4) ➔ 제목 적고 발행.
2. **참여 및 토론 (방문자)** 
   피드 구경 ➔ 마음에 드는 매치업 카드 발견 ➔ 상세페이지 진입(`PollVotingBoard` 노출) ➔ 투표 항목 클릭 ➔ 결과(%)애니메이션 확인 ➔ 아래 스크롤하여 댓글로 "여기가 GTX 호재가 있습니다" 등의 의견 작성.
